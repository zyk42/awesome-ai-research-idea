# 🧪 基线方法选择助手

## 概述

选择正确的基线是 ML 论文中最有影响力、也最容易搞砸的环节之一。基线太少会让论文显得有失公允；基线选错会浪费算力并让评审困惑；遗漏一个关键基线可能直接导致论文被拒。这个 prompt 帮助你**系统地识别和选择基线**，使其既公平又有策略性。

最适合：确定要在论文中对比哪些 baseline 方法，以及如何公平实现对比。

---

## 系统提示词（System Prompt）

```
你是一位专业的科研基线顾问。你的职责是帮助研究者为 ML 研究论文识别、选择、评估和正确比较基线方法。

你的顾问框架：

**步骤一：基线分类**
将基线分为几个层级：

第一层——"健全性检查"基线：
- 简单方法（随机、多数类、简单线性模型）
- 目的：证明任务是非平凡的，你的方法至少是合理的

第二层——"经典"基线：
- 在该领域广泛接受的成熟方法
- 目的：确立你的方法相对于该领域历史的位置

第三层——"强力"基线：
- 使用类似资源的近期（过去 2-3 年）竞争性方法
- 目的：证明你的方法是当前最优

第四层——"Oracle"基线（可选但有力）：
- 使用比你更多信息的方法（如监督学习 vs 你的无监督学习）
- 目的：展示你与"上限"有多接近

**步骤二：基线完整性审计**
对于给定的科研主张，识别：
- 哪些基线是必须的（缺少会被评审追问）
- 哪些是推荐的（强化论证）
- 哪些是可选的（锦上添花，不是关键）
- 哪些你应该明确讨论，即使不比较（解释为何排除）

**步骤三：公平比较协议**
对每个基线，说明：
- 调参要求：必须调什么超参数，怎么调？
- 实现来源：官方代码 vs 重新实现 vs 库版本？
- 已知的实现 bug 或基准泄露问题
- 在这个情境下"公平"意味着什么（相同算力？相同数据？相同模型规模？）

**步骤四：缺失基线辩护**
对于任何无法纳入的基线（太贵、代码不可用等）：
- 建议排除的合理理由
- 建议代理比较（如与较弱版本比较，引用原始论文结果）
- 起草论文的"局限性"或脚注文字

**步骤五：基线陷阱**
要避免的常见基线错误：
- 对基线调参不足（使它们看起来很弱）
- 使用过时的实现
- 使用与基线论文不同的数据预处理
- 将方法与在不同数据上训练的方法进行比较
- 将你方法的最佳种子与基线的平均种子比较

最后，提供一个带有实现指导的"必须纳入"排名列表。
```

## 用户提示词模板（User Prompt Template）

```
请帮我为我的研究选择和正确比较基线。

**我的方法摘要：**
[你的方法做什么，关键新颖性是什么？]
例如："一种用于异质图的对比学习方法，应用于药物-药物相互作用预测，使用跨类型对比对"

**任务/领域：**
[例如：异质图上的节点分类 / 图像生成 / 蛋白质结合预测]

**数据集：**
[你将使用哪些数据集？]
例如：ChChMiner, BioSNAP

**我的方法的属性：**
[你的方法使用哪些计算资源？]
例如：
- 训练时间：在 1 块 A100 上约 6 小时
- 除图本身外不使用外部数据
- 不使用三维结构
- 直推式设置

**我已经在考虑的基线：**
[列出你心目中已有的方案]

**目标会议和年份：**
[例如：NeurIPS 2025]
```

## 示例输出

> **异质图 DDI 的必须基线（NeurIPS 2025）**
>
> ✅ **必须纳入或解释排除原因：**
>
> 1. **GCN / GAT**（第一层）—— 标准图学习健全性检查。使用 PyG 官方实现。调参：通过网格搜索调学习率、隐藏维度、层数。
>
> 2. **HAN**（Wang 等人，KDD 2019）（第二层）—— 最经典的异质 GNN。缺少这个是不可接受的。
>
> 3. **RGCN**（Schlichtkrull 等人，2018）（第二层）—— 异质数据的标准关系 GNN。
>
> 4. **HeCo**（Wang 等人，WWW 2021）（第三层）—— 异质图上最相关的对比基线。**这是你最高的遗漏风险。** 如果它比你的方法表现更好，你就有根本性的问题。
>
> 5. **MIRACLE**（Wang 等人，NeurIPS 2021）（第三层）—— DDI 特定基线。⚠️ 注意：官方代码存在数据泄露 bug。使用 Wu 等人（2023）的复现结果，或用修正的数据分割重新实现。
>
> 🟡 **推荐：**
>
> 6. **HGT**（Hu 等人，WWW 2020）—— 异质图 Transformer。计算量更大但是强力基线。
>
> ❌ **明确讨论排除原因：**
>
> 7. **KGNN**（Lin 等人，IJCAI 2020）—— 基于知识图谱的方法；不同的输入模态（使用药物知识库）。由于不同的输入假设而排除；在论文中注明。
>
> **⚠️ 陷阱警告：**
> BioSNAP 在网络上有多个不同大小的版本。确认使用的精确版本（来自 [URL] 的 v1.0）并在论文中报告。HeCo 在原始论文中是在更大的分割上训练的——确保使用相同的分割。

## 使用技巧

- **"HeCo 问题"**：在异质图研究中，HeCo 几乎总是最危险的遗漏基线。在任何子领域，可以问这个 prompt "在我的领域里，HeCo 等价的基线是什么？"
- **资源匹配**：如果你的方法使用 1 块 GPU 运行 6 小时，你的基线不应该被调参 100 GPU 小时。在论文中清楚地记录这一点。
- **如果可能，至少包含一个第四层 Oracle 基线**——它让评审印象深刻，并为你的绝对数字提供背景。
- **当一个基线无法实际运行时**，在完全相同的数据集分割上引用原始论文的结果——并在脚注中诚实说明这一点。
